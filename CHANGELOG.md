# Changelog

## v2.2

- 修复 v2.1 的安装链路缺陷：用户只更新 SKILL.md 时，插件不会自动进入 Hermes 运行时。
- 新增 `scripts/install_relationship_map_feishu_card.py`，用于安装/启用 `relationship-map-feishu-card` 插件。
- 新增硬规则：插件未启用前，不得继续打开人脉地图，不得输出任何伪按钮。
- 新增 `scripts/test_v22_package_contract.py`，验证 v2.2 包含安装器、启用要求和重启要求。
- 明确验收标准：飞书消息必须为 `interactive`，按钮 value 必须包含 `relationship_map_action`。


## v2.1

- 修复 v2.0 的关键缺陷：禁止用 `[查看详情]` 等文本伪装飞书交互按钮。
- 新增系统级 companion 插件 `plugins/relationship-map-feishu-card`，通过 `pre_gateway_dispatch` 在模型回复前接管“打开人脉地图”。
- 插件直接发送飞书原生 `msg_type=interactive` 卡片；没有流式卡片插件时也能显示真实按钮。
- 新增 `/card` 回调处理：点击“查看详情 / 返回列表 / 新增联系人 / 搜索筛选”后继续发送真实交互卡片。
- 新增 `scripts/test_relationship_map_feishu_plugin.py`，验证消息类型为 interactive、存在 button、没有 `[查看详情]` 伪按钮。
- 明确未启用 companion 插件时不得声称已具备真实飞书卡片能力。

正式链接：https://raw.githubusercontent.com/xyxw1234-bot/node-engine-relationship-map-skill/main/节点引擎-人脉地图/SKILL.md

## v2.0 中文短链结构

- 将正式安装入口调整为：https://raw.githubusercontent.com/xyxw1234-bot/节点引擎-人脉地图/main/SKILL.md
- 将完整 Skill 包同步到仓库根目录，支持 raw 链接直接指向 `/main/SKILL.md`。
- 强化“只发链接、不带话术”自动识别、审查、安装、检查更新规则。
- 保留飞书流式卡片优先、飞书原生交互卡片兜底、极简纯文本最终兜底策略。

# 更新记录

## 2.0

- 新增“只发 raw SKILL.md 链接即可识别为安装/更新”的说明与安装入口。
- 补充 INSTALL.md，便于用户转发给自己的节点引擎。
- 明确安装完成只给极简回复，避免飞书刷长 Markdown。

## 1.9.0

- 新增飞书原生卡片配套层：空状态、列表、详情、确认、示例和二级页面卡片。
- 明确流式卡片优先；无流式卡片时用飞书原生 interactive card 兜底。
- 复杂能力模块统一进入二级页面，不再平铺成长文档。

## 1.8.1

- 新增飞书友好输出规范：安装和使用场景默认短摘要，不直接贴完整 Markdown。
- 明确支持卡片能力时优先用飞书交互卡片/流式卡片；不支持时退化为短纯文本。
- 说明 Skill 本身不能强制网关发送卡片，真实卡片化依赖 Hermes 飞书适配器或插件能力。

## 1.8.0

- 新增自适应关系维度：基础骨架稳定，具体字段/指标/场景/筛选由 AI 根据用户输入动态生成。
- 内置投资人、政府/协会、学校客户、渠道、专家、供应链等预设角色包，但都可由用户扩展或关闭。
- 新增维度必须有触发依据，并先作为待确认建议，不允许无依据硬填。

## 1.7.2

- 新增用户自定义扩展边界：用户可自主增加模块、字段、指标、场景和筛选。
- 推荐扩展目录 `$HERMES_HOME/data/relationship-map/extensions/`，避免官方更新覆盖用户自定义内容。
- 明确自定义不能突破语义触发、写入确认、敏感信息和事实/推断分层底线。

## 1.7.1

- 明确 Hermes 原生运行边界：不单独部署服务器，不依赖外部数据库服务。
- 默认数据目录改为当前 Hermes profile 下的 `$HERMES_HOME/data/relationship-map/`，无环境变量时回退 `~/.hermes/data/relationship-map/`。
- SQLite 仅作为 Hermes profile 内部数据保险箱文件。

## 1.7.0

- 新增 SQLite 本地数据保险箱、schema migration、备份、审计日志。
- 新增平台中立卡片状态机适配层，保持列表/详情/返回的简洁 UX。
- 新增 5000 联系人 / 10000 轮存储层与卡片层端到端测试。
- 后台复杂度增强，但用户侧仍只看到打开、筛选、查看详情、返回、确认更新。

## 1.6.0

- 加强运行层：疑问句走确认、城市打开请求自动带筛选。
- 扩展敏感字段覆盖，一级列表防泄露压力测试覆盖电话、微信、地址、证件、财务备注、私密评价。
- 指标必须带 evidence〔依据〕，扩展测试强制校验。
- 写入安全覆盖删除、合并、批量导入、敏感标签、风险和信任评估；非法字段拒绝。
- 通过 5000 联系人 / 10000 轮扩展压力测试后才建议商用前试运行。

## 1.5.0

- 发布前可运行 `python3 relationship-map/scripts/run_runtime_stress_tests.py --contacts 500 --rounds 2000`。
