# 飞书友好输出规范

## 问题

如果用户没有安装飞书流式卡片或折叠卡片能力，长 Markdown 会在飞书里显示成大段裸文本，出现 `##`、`**`、`---`、表格源码等，视觉体验很差。

## 规范

- 不直接输出完整 Skill 内容。
- 安装成功只给极简确认和 1—2 个试用句。
- 打开列表只显示卡片摘要，不刷完整联系人。
- 详情页只显示当前联系人，不附带长规则说明。
- 如果平台支持飞书 interactive card 或 streaming card，优先用卡片。
- 如果不支持，退化为无 Markdown 裸标记的短纯文本。

## 边界

Skill 本身不能强制飞书网关发送卡片。真正卡片化需要 Hermes 飞书适配器、流式卡片插件或交互卡片发送能力支持。
