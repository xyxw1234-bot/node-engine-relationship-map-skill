# v2.2 最严格 LOP〔本地操作规程〕

本 LOP 是人脉地图 v2.2 发布前强制门禁。任何一项失败，都不得向用户或 GitHub 声称“已完成”。

## 0. 红线

- 不得把 `SKILL.md` 更新说成插件已安装。
- 不得把文档、规则、渲染器、本地代码说成生产可用。
- 不得输出 `[查看详情]`、`新增联系人｜搜索` 这类伪按钮。
- 飞书卡片必须是 `msg_type=interactive`。
- 按钮 value 必须包含 `relationship_map_action`。
- 插件未安装/启用/重启生效前，不得继续打开人脉地图。

## 1. 全局文件同步门禁

检查范围：

- `SKILL.md`
- `节点引擎-人脉地图/SKILL.md`
- `relationship-map/SKILL.md`
- `README.md`
- `INSTALL.md`
- `CHANGELOG.md`
- `references/`
- `relationship-map/references/`
- `plugins/relationship-map-feishu-card/`
- `scripts/`
- `relationship-map/scripts/`
- `templates/`
- `relationship-map/templates/`

必须确认：

- 对外版本均为 v2.2。
- 插件 manifest 为 `version: 2.2.0`。
- 中文路径、根路径、relationship-map 子路径三份核心 Skill 同步。
- README/INSTALL/CHANGELOG 均说明插件安装、启用、重启、验收。

## 2. 陈旧污染清理门禁

必须扫描并处理：

- `__pycache__`、`.pyc`、临时文件、测试残留数据库。
- 正向伪按钮示例。
- 仍暗示“只发链接即可完成插件安装”的说明。
- 旧版中文命名污染。
- 插件版本号残留 `2.1.0`。
- 旧 frontmatter 版本 `version: 2.0` / `version: 2.1`。

允许保留：

- CHANGELOG 中作为历史缺陷说明出现的 v2.0/v2.1。
- 明确写着“禁止、不得、失败、伪按钮”的反例。

## 3. 隔离环境验收门禁

必须创建临时 `HERMES_HOME`，模拟真实用户：

1. 安装插件到 `$HERMES_HOME/plugins/relationship-map-feishu-card`。
2. 写入 `config.yaml` 并启用插件。
3. 写入模拟人脉数据库。
4. 模拟飞书用户发送“打开人脉地图”。
5. 验证被 `pre_gateway_dispatch` 拦截，模型不参与伪卡片生成。
6. 验证发送 `msg_type=interactive`。
7. 验证有真实 `button`。
8. 验证“查看详情”按钮 value 包含 `relationship_map_action=detail`。
9. 模拟 `/card` 回调。
10. 验证详情页仍为 `interactive`，且包含“返回列表”按钮。
11. 验证“比如打开人脉地图怎么设计”不会误触发。

## 4. 安装器门禁

必须验证：

- `scripts/install_relationship_map_feishu_card.py` 存在。
- 安装器包含 `--enable`。
- 安装器输出 `restart_required=true`。
- 老版本 Hermes CLI 不支持时，至少能写入配置兜底。
- 安装器不能吞掉失败；失败时必须返回非零状态。

## 5. 压力与边界测试门禁

必须通过：

- `scripts/test_v22_package_contract.py`
- `scripts/test_relationship_map_feishu_plugin.py`
- `scripts/test_feishu_card_renderer.py`
- `scripts/run_runtime_stress_tests.py`

## 8. GitHub Release 页面门禁

发布到 GitHub 不只包括 main 分支和 git tag。必须同时创建或更新 GitHub Releases 页面上的 release 对象，并确认右侧 Releases 最新版本显示为 v2.2。

强制检查：

- `GET /repos/xyxw1234-bot/node-engine-relationship-map-skill/releases/latest` 返回 `tag_name == v2.2`。
- `GET /repos/xyxw1234-bot/node-engine-relationship-map-skill/releases/tags/v2.2` 存在。
- 仓库 About 右侧 Releases 不得继续显示 v2.0 Latest。
- 如未通过，运行 `scripts/create_github_release_v22.py` 创建/更新 release 对象。
